0. 前言

本篇内容是对前端web安全知识的学习，了解常遇到的前端攻击及防范方法。

1. XSS攻击

  • 攻击

XSS攻击全称跨站脚本攻击。

恶意攻击者向Web页面里注入恶意script代码，用户浏览这些网页时会执行其中的恶意代码，可对用户进行盗取cookie信息、会话劫持等各种攻击。

XSS攻击有两种，一种是通过修改浏览器URL导致脚本被注入到页面，另一种是通过输入框将脚本代码注入数据库。

  • 防范
  1. 过滤用户的输入信息，禁止用户在输入的过程中输入 “<”, “>”, 引号, “$”, “_”。
  2. 核心的用户身份标示或token保存在Cookie中，Cookie中一定要加“HTTP Only”在结尾，保证只有在 html 操作时才能将Cookie中的内容发送出去，在 JS 中无法获得用户的Cookie信息。

2. CSRF攻击

  • 攻击

CSRF中文名为跨站请求伪造。

它与 XSS 非常不同，XSS 利用站点内的信任用户，而 CSRF 则通过伪装来自受信任用户的请求来利用受信任的网站。与 XSS 攻击相比，CSRF 攻击往往不大流行（因此对其进行防范的资源也相当稀少）和难以防范，所以被认为比 XSS 更具危险性。

  • 防范
  1. 减少在 cookie 中存储客户核心内容比如用户的 token、ID、access_token 等
  2. GET 请求不对数据进行修改
  3. 不让第三方网站访问到 Cookie
  4. 阻止第三方网站请求接口
  5. 请求时附带验证信息，比如验证码或者 Token

3. 点击劫持

点击劫持是一种视觉欺骗的攻击手段。攻击者将需要攻击的网站通过iframe嵌入自己的网页中，并将iframe设置为透明，然后诱使用户在该页面上进行操作，此时用户将在不知情的情况下点击透明的iframe页面。

防御方法：

还是让后端解决，使用一个HTTP响应头——X-Frame-Options。X-Frame-Options可以说是为了解决点击劫持而生的，它有三个可选的值：

  • DENY：浏览器会拒绝当前页面加载任何frame页面
  • SAMEORIGIN：frame页面的地址只能为同源域名下的页面
  • ALLOW-FROM origin：允许frame加载的页面地址
